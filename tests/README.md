# اختبارات ضمان الجودة (QA)

حزمة اختبارات آلية تُشغّل التطبيق في متصفح حقيقي (Chromium عبر Playwright) وتتحقق من:

- **صحة المنطق والنتائج المحسوبة** — احتساب نتيجة التدقيق، المؤشرات، تصنيف درجات الحرارة، دوال التاريخ، شارات الصلاحية.
- **قاعدة المواصفات** — مطابقة الإجراءات التصحيحية/الوقائية ووجود المعايير السعودية والخليجية والعالمية.
- **الذكاء الاصطناعي (الوضع المحلي)** — توليد الإجراءات، تقييم البنود، رصد المخالفات من الملاحظة، وطلب المفتاح عند غيابه.
- **تدفقات المستخدم الكاملة** — التفتيش وإنشاء المخالفات تلقائيًا، قراءة حرارة مخالفة، CAPA بالذكاء الاصطناعي، الرصد بالتصوير، جدولة التنظيف، الإعدادات، وتصدير/استيراد البيانات.
- **سلامة التشغيل** — عدم وجود أي أخطاء في كونسول المتصفح طوال الاختبارات.

## التشغيل

```bash
# 1) ثبّت الاعتمادات (Playwright)
npm install
npx playwright install chromium

# 2) شغّل خادمًا محليًا للتطبيق على المنفذ 8123
python3 -m http.server 8123 &

# 3) شغّل الحزمة
npm test          # أو: node tests/qa.js
```

ينهي البرنامج بكود خروج `0` عند نجاح جميع الاختبارات، و`1` عند وجود أي فشل.

## التشغيل الآلي (CI)

يُشغّل سير العمل `.github/workflows/ci.yml` هذه الحزمة آليًا على **كل دفعة وكل Pull Request** عبر GitHub Actions.

> آخر تشغيل موثّق: **50/50 ناجح، 0 فاشل، بدون أخطاء كونسول.**
