# تحويل النظام إلى SaaS سحابي (Supabase)

يحوّل هذا الدليل التطبيق من «وضع عرض محلي» إلى **منصة SaaS متعددة المستأجرين**: كل منشأة معزولة ببياناتها وحسابها، مع تسجيل دخول حقيقي، ومزامنة سحابية، ونموذج اشتراك (خطط).

> الواجهة وكل الوحدات تبقى كما هي — يتغيّر فقط مصدر البيانات (سحابي بدل محلي). يُفعَّل الوضع السحابي تلقائيًا عند ضبط `assets/js/config.js`.

---

## 1) أنشئ مشروع Supabase
1. ادخل إلى <https://supabase.com> وأنشئ حسابًا ومشروعًا جديدًا (الخطة المجانية تكفي للبدء).
2. من **Project Settings → API** انسخ:
   - **Project URL** (مثل `https://abcd.supabase.co`)
   - **anon public key**

## 2) أنشئ قاعدة البيانات
1. افتح **SQL Editor** في لوحة Supabase.
2. الصق محتوى الملف [`schema.sql`](./schema.sql) كاملًا واضغط **Run**.
3. يُنشئ هذا: جداول المنشآت والعضويات وكل وحدات النظام، مع **عزل صارم للبيانات (RLS)** ودالة إنشاء المنشأة.

## 3) إعداد المصادقة
- من **Authentication → Providers → Email**: فعّل تسجيل الدخول بالبريد.
- للتجربة السريعة: عطّل **Confirm email** (Authentication → Settings) ليدخل المستخدم مباشرة بعد التسجيل. (في الإنتاج أبقِه مفعّلًا.)

## 4) اربط التطبيق
انسخ `assets/js/config.example.js` إلى `assets/js/config.js` وأدخل مفاتيحك:
```js
window.SAAS = {
  url: 'https://abcd.supabase.co',
  anonKey: 'eyJ...your-anon-key...',
};
```
بمجرد ضبط القيمتين، يتحوّل التطبيق تلقائيًا إلى الوضع السحابي: شاشة تسجيل/دخول حقيقية بدل أدوار العرض.

## 5) جرّب
- افتح التطبيق → **أنشئ منشأة جديدة** (بريد + كلمة مرور + اسم المنشأة).
- يُنشأ حساب ومنشأة معزولة على خطة **تجريبية (14 يومًا)**.
- كل ما تسجّله (تفتيش، حرارة، شهادات…) يُحفظ سحابيًا ويُزامَن، ويظهر فقط لمنشأتك.

---

## كيف يعمل العزل (Multi-tenant)
- كل سجل مرتبط بـ `org_id`، وسياسات **Row-Level Security** تمنع أي مستخدم من رؤية بيانات منشأة أخرى — يُفرض على مستوى قاعدة البيانات نفسها، لا الواجهة.
- بيانات كل وحدة تُخزَّن ككائن `JSONB` بنفس شكل التطبيق، فالتكامل بسيط ومتين.

## الخطط والاشتراك
- الخطط وحدودها مُعرّفة في `assets/js/cloud.js → PLAN_LIMITS` (تجريبية/أساسية/احترافية/مؤسسية).
- صفحة **الاشتراك** داخل التطبيق تعرض خطة المنشأة وحالتها وأيام التجربة، وميزة الذكاء الاصطناعي تُقيَّد حسب الخطة.
- **الفوترة (الدفع الإلكتروني):** مؤجّلة حسب الاتفاق. لإضافتها لاحقًا:
  1. أنشئ Edge Function تستقبل Webhook من بوابة الدفع (Moyasar/Tap/Stripe).
  2. عند نجاح الدفع، حدّث `organizations.plan` و`subscription_status`.

## النشر
استضِف الملفات الثابتة على أي مزوّد (Netlify/Vercel/Cloudflare Pages/خادمك) واربط دومينك. لا حاجة لخادم خاص — Supabase يتولّى البيانات والمصادقة.

## المرحلة الثانية: الفريق والأدوار + الدفع

### أ) الفريق والأدوار
1. شغّل [`02_team_and_billing.sql`](./02_team_and_billing.sql) في **SQL Editor** (بعد `schema.sql`).
2. يضيف: ملفات التعريف، الدعوات، الأدوار (مالك/مدير/مفتش)، ودوال الدعوة والقبول والإزالة.
3. داخل التطبيق → صفحة **الفريق والأدوار**: المالك/المدير يدعو أعضاء بالبريد ويحدد الدور؛ ينضم العضو تلقائيًا عند تسجيله بنفس البريد.

### ب) الدفع (Moyasar)
1. أنشئ حساب <https://moyasar.com> واحصل على **Secret Key**.
2. انشر دالتي الحافة:
   ```bash
   supabase functions deploy create-checkout
   supabase functions deploy moyasar-webhook --no-verify-jwt
   ```
3. اضبط الأسرار:
   ```bash
   supabase secrets set MOYASAR_SECRET_KEY=sk_... APP_URL=https://your-app-url \
     MOYASAR_WEBHOOK_SECRET=your-random-secret SUPABASE_SERVICE_ROLE_KEY=service_role_key
   ```
4. في لوحة Moyasar → **Webhooks**: أضِف رابط `moyasar-webhook` لحدث `payment_paid`، مع ترويسة `x-webhook-secret`.
5. داخل التطبيق → صفحة **الاشتراك** → «اختر الخطة» → يُحوَّل العميل لصفحة دفع Moyasar الآمنة، وعند نجاح الدفع تُفعَّل خطته تلقائيًا عبر الـ Webhook.

> الأسعار في `saas/functions/create-checkout/index.ts` (بالهللة). عدّلها حسب باقاتك.

### ج) الذكاء الاصطناعي عبر بوابة آمنة (claude-proxy) — مُوصى به
بدل إدخال مفتاح Claude في المتصفح (حيث يمكن أن يراه المستخدم)، تمرّ استدعاءات الذكاء
الاصطناعي عبر دالة حافة تحفظ المفتاح في الخادم وتتحقق من هوية المستخدم وأن خطته تتيح الميزة.

1. انشر الدالة:
   ```bash
   supabase functions deploy claude-proxy
   ```
2. اضبط مفتاح Anthropic كسرّ على الخادم (احصل عليه من <https://console.anthropic.com>):
   ```bash
   supabase secrets set ANTHROPIC_API_KEY=sk-ant-...
   # اختياري: ANTHROPIC_MODEL=claude-opus-4-8  MAX_TOKENS_CAP=3000
   ```
3. هذا كل شيء — في الوضع السحابي يكتشف التطبيق البوابة تلقائيًا: تعمل ميزات التحليل
   والتصوير وتقدير التغذية للعملاء الذين تتيح خطتهم الذكاء الاصطناعي (trial/pro/enterprise)
   **دون أن يُدخل أي عميل مفتاحًا**، ودون أن يظهر المفتاح في المتصفح إطلاقًا.

> الوضع المحلي (بدون Supabase) ما زال يدعم إدخال مفتاح خاص من صفحة الإعدادات للتجربة الفردية.

## ملاحظات
- مفتاح **anon** عام بطبيعته وآمن للواجهة؛ الحماية الفعلية عبر RLS.
- مفتاح **service_role** يُستخدم فقط داخل دالة الحافة (الخادم) ولا يُكشف للعميل أبدًا.
- مفتاح Claude API للذكاء الاصطناعي يُدار كما هو (لكل منشأة، أو عبر وكيل خلفي لاحقًا).
- وضع العرض المحلي يبقى متاحًا (config فارغ) للتجارب والعروض دون إنترنت.
